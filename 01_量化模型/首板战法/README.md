# 强度首板战法模型

## 策略概述

本模型用于识别"强度首板战法"机会，核心逻辑：

1. **连续涨停1天**：非连板（涨停数=1）
2. **早盘买入**：集合竞价附近买入
3. **次日卖出**：收盘前全部卖出
4. **仓位控制**：每次买入50%（五层仓）
5. **综合评分**：根据K线趋势、未来走法、情绪、基本面综合分析

## 筛选条件

### 硬性条件
- ✅ 连续涨停天数为1天（非连板）
- ✅ 排除ST股票
- ✅ 排除科创板（688开头）
- ✅ 排除北交所（8开头）
- ✅ **已移除**：股价<100元、市值<200亿的限制

### 综合评分系统（100分制）

#### 1. K线趋势分析（25%权重）

| 评分项 | 权重 | 说明 |
|--------|------|------|
| 均线排列 | 30分 | MA5>MA10>MA20=30分；MA5>MA10=20分；MA10>MA20=15分；其他=5分 |
| 近期涨幅 | 25分 | >15%=25分；>5%=20分；>0%=15分；>-5%=10分；<-5%=5分 |
| 成交量趋势 | 25分 | >2倍=25分；>1.5倍=20分；>1倍=15分；其他=10分 |
| 突破形态 | 20分 | 突破20日高点=20分；站上均线=15分；未突破=10分 |

#### 2. 未来走法分析（25%权重）

| 评分项 | 权重 | 说明 |
|--------|------|------|
| 上涨空间 | 30分 | 距20日均线<5%=30分；5%-10%=20分；10%-20%=10分；>20%=5分 |
| 支撑位强度 | 35分 | 距10日最低<3%=35分；3%-8%=25分；8%-15%=15分；>15%=5分 |
| 趋势延续性 | 35分 | 趋势得分≥80=35分；≥60=25分；≥40=15分；<40=5分 |

#### 3. 情绪分析（25%权重）

| 评分项 | 权重 | 说明 |
|--------|------|------|
| 涨停原因 | 30分 | 热点题材=30分；一般原因=20分；无原因=10分 |
| 换手率 | 30分 | 5%-15%=30分；>15%=25分；3%-5%=20分；<3%=10分 |
| 封单强度 | 40分 | >10亿=40分；5-10亿=30分；2-5亿=20分；<2亿=10分 |

**热点关键词**：AI、新能源、芯片、半导体、汽车、医药、消费、科技、数字经济

#### 4. 基本面分析（25%权重）

| 评分项 | 权重 | 说明 |
|--------|------|------|
| 市值 | 40分 | <50亿=40分；50-100亿=30分；100-200亿=20分；>200亿=5分 |
| 股价 | 30分 | <10元=30分；10-20元=25分；20-50元=20分；50-100元=15分；>100元=5分 |
| 市盈率 | 30分 | <20=30分；20-40=25分；40-80=15分；>80=5分；无数据=15分 |

## 使用方法

### 基本运行

```bash
# 使用默认参数（今天，前2只）
python up.py

# 指定日期和数量
python up.py --date 2024-04-24 --top-n 2

# 自定义输出文件
python up.py --date 2024-04-24 --top-n 2 --output my_result.json
```

### 参数说明

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--date` | 分析日期，格式：YYYY-MM-DD | 今天 |
| `--top-n` | 返回前N只股票 | 2 |
| `--output` | 输出JSON文件路径 | ./strength_limit_up_result.json |

## 交易策略

### 今日操作

**买入时机**：早盘集合竞价附近（9:25）

**买入仓位**：50%（五层仓）

**目标收益**：3%-8%

**止损策略**：涨停板最低价

### 明日操作

1. **选股**：继续用模型选出新的强度票
2. **买入**：用剩余50%仓位在集合竞价附近买入
3. **卖出**：无论盈亏，收盘前必须全部卖出
4. **执行T+1策略**：不隔夜持有

## 技术实现

### 数据源

- **涨停池**：`akshare.stock_zt_pool_em()` - 东方财富涨停数据
- **历史K线**：`akshare.stock_zh_a_hist()` - 前复权日线数据
- **实时行情**：`akshare.stock_zh_a_spot_em()` - 实时行情数据

### 核心函数

- `get_zt_pool_with_retry()` - 获取涨停池（带重试）
- `get_stock_history_with_retry()` - 获取历史数据（带重试）
- `get_stock_fundamentals_with_retry()` - 获取基本面数据（带重试）
- `check_single_day_limit_up()` - 检查是否为连续涨停1天
- `calculate_simple_score()` - 简单评分方案（当无法获取历史数据时）
- `calculate_composite_score()` - 计算综合强度得分（智能选择评分方案）
- `analyze_k_line_trend()` - 分析K线趋势
- `analyze_future_outlook()` - 分析未来走法
- `analyze_sentiment()` - 分析市场情绪
- `analyze_fundamentals()` - 分析基本面

### 评分方案

#### 完整评分（100分制）
当能获取历史数据时使用：
- **K线趋势分析**（25分）：均线排列、近期涨幅、成交量趋势、突破形态
- **未来走法分析**（25分）：上涨空间、支撑位强度、趋势延续性
- **情绪分析**（25分）：涨停原因、换手率、封单强度
- **基本面分析**（25分）：市值、股价、市盈率

#### 简单评分（100分制）
当无法获取历史数据时使用：
- **涨停数据**（50分）：成交额、换手率、涨停幅度
- **热点题材**（25分）：涨停原因分析
- **基本信息**（25分）：股价、市值

### 网络容错

- 涨停池查询：最多重试3次
- 历史数据查询：最多重试3次
- 基本面查询：最多重试3次
- 股票代码自动标准化处理
- **智能评分方案**：
  - 能获取历史数据 → 完整评分（100分制）
  - 无法获取历史数据 → 简单评分（100分制）
  - 确保所有涨停股票都能得到评分
- 失败时不使用演示数据，返回真实结果

### 代码优化

1. **股票代码标准化**
   - 自动移除前缀（如sz000001 → 000001）
   - 兼容不同格式输入

2. **错误处理**
   - 捕获所有异常类型
   - 详细错误信息输出
   - 不影响其他股票分析

3. **数据验证**
   - 检查历史数据量（至少10条）
   - 验证数据完整性
   - 处理缺失值和异常值

4. **评分优化**
   - 智能选择评分方案
   - 确保所有股票都能评分
   - 输出评分类型标识

## 常见问题

### Q1: 程序报错"无法获取历史数据"

**原因**：
- 某些股票代码格式问题（如创业板股票300xxx）
- 数据源临时不可用
- 网络连接不稳定

**解决方案**：
- ✅ 程序已自动优化股票代码格式转换
- ✅ 内置3次重试机制
- ✅ **自动切换到简单评分方案**，确保股票仍然可以得到评分
- ✅ 不会跳过股票，所有涨停股票都会被评分
- ✅ 建议在网络稳定时运行以获得完整评分

### Q2: 程序报错"KeyError: 'future'"或其他KeyError

**原因**：
- 某些分析组件数据不完整
- 字典键名访问错误

**解决方案**：
- ✅ 程序已修复，使用安全的字典访问方法（.get()）
- ✅ 检查键是否存在后再访问
- ✅ 提供默认值避免崩溃
- ✅ 如果仍有问题，请检查网络连接

### Q3: 程序报错"获取涨停池失败"

**原因**：网络连接不稳定或数据源临时不可用

**解决方案**：
- ✅ 程序已内置重试机制（最多3次）
- ✅ 如果仍然失败，程序会返回空结果
- ✅ **不会使用演示数据**，确保结果真实性
- ✅ 建议稍后重试或检查网络连接

### Q4: 找不到符合条件的股票

**原因**：市场环境不符合策略条件或硬性筛选过严

**解决方案**：
- ✅ 程序会返回空结果，不使用演示数据
- ✅ 检查以下可能原因：
  1. 市场环境不符合策略条件
  2. 硬性筛选条件过严（股价<100，市值<200亿）
  3. 网络连接问题导致数据获取失败
  4. 当前无符合条件的首次涨停股票
- ✅ 可以适当调整筛选条件
- ✅ 建议在其他交易日运行

### Q3: 如何提高评分准确性

**方法**：
- 在交易日收盘后运行（数据最完整）
- 确保网络连接稳定
- 多日观察，建立评分基准
- 结合人工判断调整权重

### Q4: 模型选出的股票如何交易

**建议**：
- 严格按照交易策略执行
- 集合竞价附近买入，不追高
- 设置好止损位
- 次日收盘前必须清仓
- 不要贪婪，落袋为安
