# akshare 导入问题 - 永久解决方案

## 问题描述

```
ModuleNotFoundError: No module named 'akshare'
```

## 问题根因

- Python 和 pip 安装在不同的路径
- akshare 被安装到了用户目录 `~/.local/lib/python3.13/site-packages/`
- 系统 Python 默认找不到用户安装的包

## 解决方案

### 1. 自动修复（已在代码中实现）✅

在所有量化模型文件的**开头**添加路径设置代码：

```python
import sys
import os

# 添加用户 site-packages 路径（确保 akshare 能被找到）
user_site_packages = os.path.expanduser('~/.local/lib/python3.13/site-packages')
if user_site_packages not in sys.path:
    sys.path.insert(0, user_site_packages)
```

**已修改的文件**：
- `01_量化模型/首板战法/stock.py`
- `01_量化模型/首板战法/up.py`
- `01_量化模型/三阴战法/pop3.py`
- `01_量化模型/隔夜战法/deep.py`
- `01_量化模型/隔夜战法/stock_quant_select.py`

### 2. 使用初始化脚本（推荐）

```bash
cd /workspace/projects/stock-quant-select/01_量化模型/首板战法

# 自动检查并安装依赖，然后运行程序
./init.sh stock.py
```

### 3. 手动安装依赖

如果上述方法无效，尝试手动安装：

```bash
# 方式1：使用 --user 安装（推荐）
python3 -m pip install --user akshare pandas numpy

# 方式2：全局安装
python3 -m pip install akshare pandas numpy

# 方式3：使用 pip3（可能路径不一致）
pip3 install akshare pandas numpy
```

## GitHub 推送问题

如果推送代码到 GitHub 时遇到认证失败：

```

fatal: Authentication failed
```

**解决方案**：

### 方案1：使用 HTTPS + Token（推荐）

```bash
# 重新添加远程仓库（使用 Token）
git remote set-url origin https://<YOUR_TOKEN>@github.com/shen9220/stock-quant-select.git

# 或者重新配置
git remote add origin https://<YOUR_TOKEN>@github.com/shen9220/stock-quant-select.git

# 推送
git push origin main
```

### 方案2：使用 SSH

```bash
# 检查是否有 SSH Key
ls -la ~/.ssh/

# 生成 SSH Key（如果不存在）
ssh-keygen -t ed25519 -C "1436114007@qq.com"

# 添加到 GitHub
# Settings > SSH and GPG keys > New SSH key

# 切换到 SSH

# 推送
git push origin main
```

2. **权限问题**：如果遇到权限错误，使用 `--user` 参数

