## 📝 文件命名规范

### Python 文件
- 使用小写字母和下划线：`run_models.py`
- 策略文件：`首板战法/up.py`

### Shell 脚本
- 使用下划线：`启动模型.sh`
- 可执行文件添加 `+x` 权限

### Markdown 文档
- 使用下划线或连字符：`项目必读.md`
- 命名清晰，见名知意

### JSON 文件
- 使用下划线：`test_report.json`
- 按日期或功能命名

## ⚙️ 新增文件规则

### 新增量化模型

1. 在 `01_量化模型/` 下创建新目录
2. 目录名与模型名称一致
3. 包含：主程序 + README.md + 配置

**示例**:
```
01_量化模型/
└── 新策略/
    ├── main.py
    ├── config.json
    └── README.md
```

### 新增文档

1. 按类别放入 `04_文档说明/` 对应子目录
2. 命名清晰，便于查找
3. 更新相关索引

**示例**:
```
04_文档说明/
└── 使用指南/
    └── 新功能说明.md
```

### 新增配置

1. 放入 `05_配置文件/`
2. 如果敏感，添加到 .gitignore
3. 创建 .env.example 模板
