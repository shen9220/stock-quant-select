# 📐 项目规范

## 目录编号规则

所有主目录使用两位数字前缀，方便排序和查找：

| 编号 | 目录 | 用途 |
|------|------|------|
| 00 | 00_项目必读/ | 规则和注意事项（必须最先阅读） |
| 01 | 01_量化模型/ | 核心量化策略 |
| 02 | 02_智能工具/ | 一键运行工具 |
| 03 | 03_智能体/ | 智能体程序 |
| 04 | 04_文档说明/ | 所有文档 |
| 05 | 05_配置文件/ | 配置和敏感文件 |
| 06 | 06_测试报告/ | 测试和报告 |

---

## 文件组织原则

### 1. 关联文件放在一起

**正确**:
```
01_量化模型/首板战法/
├── up.py
├── config.json
├── README.md
└── result.json
```

**错误**:
```
01_量化模型/首板战法/up.py
04_文档说明/首板战法文档.md
06_测试报告/首板战法结果.json
```

### 2. 分类明确

每个文件必须属于明确的类别，不能混淆。

### 3. 层次清晰

- 根目录：主分类（00-06）
- 二级目录：具体功能模块
- 三级目录：相关文件

## 提交规范

### Git 提交信息格式

```
<type>(<scope>): <subject>

<body>

<footer>
```

**Type 类型**:
- `feat`: 新功能
- `fix`: 修复 bug
- `docs`: 文档更新
- `style`: 代码格式
- `refactor`: 重构
- `test`: 测试
- `chore`: 构建/工具

**示例**:
```
feat(首板战法): 添加新的评分系统

- 实现了新的综合评分算法
- 添加了技术指标分析
- 优化了数据获取逻辑

Closes #123
```
