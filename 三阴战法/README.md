# 首次涨停后阴跌战法模型

## 策略概述

本模型用于识别"首次涨停后阴跌战法"机会，核心逻辑：

1. **首次涨停识别**：涨停前非连板（涨停数=1）
2. **阴跌模式**：涨停后连续3天阴跌，不破涨停底部
3. **十字星确认**：第3或4天收十字星（振幅<2%）
4. **强度排名**：综合评分排序，选出前5只最强股票

## 筛选条件

### 必须满足的条件
- ✅ 首次涨停（涨停前无连续涨停）
- ✅ 涨停后连续3天阴跌（收盘<开盘）
- ✅ 阴跌期间不破涨停底部（允许1%误差）
- ✅ 第3或4天收十字星（振幅<2%）
- ✅ 排除ST股票
- ✅ 排除科创板（688开头）
- ✅ 排除北交所（8开头）

## 强度评分系统（100分制）

| 评分项 | 权重 | 说明 |
|--------|------|------|
| 趋势连续性 | 30分 | MA5>MA10>MA20多头排列=30分；MA5>MA10=20分；其他=10分 |
| 成交量趋势 | 20分 | 近期放量=20分；温和放量=15分；缩量=10分 |
| 回调幅度 | 20分 | 回调2%-5%=20分；<2%=15分；5%-10%=10分；>10%=5分 |
| 近期涨幅 | 20分 | 近5日涨幅>10%=20分；>0%=15分；>-5%=10分；<-5%=5分 |
| 换手率 | 10分 | 5%-15%=10分；>15%=8分；3%-5%=7分；<3%=5分 |

## 使用方法

## 交易策略

### 今日操作

1. **第一名买入**
   - 时机：集合竞价附近（9:25）
   - 仓位：半仓
   - 价格：参考当前价附近
   - 止损：涨停底部价格

2. **备选方案**
   - 条件：如果第一名开盘涨幅>5%
   - 操作：买入第二名
   - 仓位：半仓

### 明日操作

1. **选股**：继续用模型选出新的强度票
2. **买入**：用剩余半仓在开盘附近买入
3. **持有**：持有时间1天
4. **目标**：博取当日买入和次日买入后的差距
5. **清仓**：无论结果如何，收盘前必须全部清仓

## 技术实现

### 数据源

- **涨停池**：`akshare.stock_zt_pool_em()` - 东方财富涨停数据
- **历史K线**：`akshare.stock_zh_a_hist()` - 前复权日线数据

### 核心函数

- `get_zt_pool_with_retry()` - 获取涨停池（带重试）
- `get_stock_history_with_retry()` - 获取历史数据（带重试）
- `check_yin_die_pattern()` - 检查阴跌模式
- `check_first_limit_up()` - 检查首次涨停
- `calculate_strength_score()` - 计算强度得分

### 网络容错

- 涨停池查询：最多重试3次
- 历史数据查询：最多重试3次
- 自动使用演示数据（当网络完全不可用时）

## 注意事项

1. **数据时效性**：确保运行时间为交易日收盘后
2. **网络环境**：需要稳定的互联网连接
3. **风险提示**：本模型仅供参考，不构成投资建议
4. **止损纪律**：严格执行止损策略
5. **仓位控制**：严格执行半仓策略，分散风险

## 常见问题

### Q1: 程序报错"Connection aborted"

**原因**：网络连接不稳定

**解决方案**：
- 程序已内置重试机制（最多3次）
- 如果仍失败，会自动使用演示数据
- 建议在网络稳定时运行

### Q2: 找不到符合条件的股票

**原因**：市场环境不符合策略条件

**解决方案**：
- 程序会自动生成演示数据
- 演示数据包含5只股票的完整分析
- 实际使用时请关注真实市场数据

### Q3: 如何提高运行速度

**方法**：
- 限制分析天数：只查询最近5天涨停
- 限制股票数量：每天最多分析50只涨停股
- 使用缓存：避免重复查询相同数据
